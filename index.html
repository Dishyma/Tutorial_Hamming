Tutorial: Código Hamming (SEC - Single Error Correction)

Principio de funcionamiento

El código Hamming es un método de corrección de errores que permite detectar y corregir errores de un solo bit mediante la adición de bits de comprobación redundantes.

Fundamentos generales:

  1. Bits de comprobación: Para una palabra de M bits de datos, se agregan K bits de comprobación donde 2^K - 1 ≥ M + K.
  2. Posiciones especiales: Los bits de comprobación se colocan en posiciones que son potencias de 2 (1, 2, 4, 8, 16...).
  3. Verificación por grupos: Cada bit de comprobación verifica un conjunto de bits según su representación binaria.
  4. Palabra de síndrome: Al leer, se recalculan los bits de comprobación y se comparan con los almacenados. La diferencia (síndrome) indica la posición del error.
Para W=8 bits de datos: 2^K - 1 ≥ 8 + K → K = 4 bits de comprobación.
Total de bits almacenados: W + K = 8 + 4 = 12 bits.

Estructura de posiciones para W=8:

Posición 121110987654321
Binario 110010111010100110000111011001010100001100100001
Contenido D8D7D6D5C8D4D3D2C4D1C2C1

Aplicación del código Hamming

PASO 1: Identificar los bits de datos y colocarlos en las posiciones correspondientes.
PASO 2: Calcular los bits de comprobación (C1, C2, C4, C8) mediante XOR según las posiciones que cada uno verifica.
Como resultado, se obtiene una palabra de 12 bits que combina los datos y los bits de comprobación.

Esta palabra se almacena en memoria y permite detectar y corregir un error de un solo bit si ocurre.

Verificación y detección de errores

Si la palabra almacenada sufre una alteración en uno de sus bits, se puede detectar el error recalculando los bits de comprobación y comparándolos con los almacenados.
PASO 1: Recalcular los bits de comprobación con los datos recibidos.
PASO 2: Comparar los bits recalculados con los almacenados para obtener el síndrome.
PASO 3: Interpretar el síndrome: su valor binario indica la posición del bit erróneo dentro de la palabra.
Si el síndrome es distinto de 0000, el número binario indica la posición del error.

Por ejemplo, un síndrome 0111 (decimal 7) indica un error en la posición 7.

Para corregirlo, basta con invertir el bit correspondiente.

Tabla resumen del proceso de detección:

Síndrome Binario Decimal Significado
0000 0000 0 No hay error
0111 0111 7 Error en posición 7
Cualquier otro XXXX 1–15 Error en posición indicada

Resumen de conceptos clave:

  • El síndrome se obtiene comparando bits de comprobación almacenados y recalculados.
  • Un síndrome distinto de 0000 indica la posición del error.
  • El código Hamming permite corregir un error simple (SEC).
  • La posición del error se obtiene interpretando el síndrome en binario.
